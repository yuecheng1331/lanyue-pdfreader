## 2026-07-15 V0.5 阶段 C2 更新

- 标签可按住文件名区域拖放到其他标签位置调整顺序；排序只影响当前运行会话，不改变最近文件顺序。

- 每个标签分别保存翻译面板快照，包括可编辑原文、译文、翻译风格、目标语言和自定义指令；切换标签会结束当前鼠标选区和在途翻译，返回标签后恢复其文本上下文，不会把 A 文档的译文显示到 B 文档。
- 关闭非当前标签只释放对应 Worker 文档；关闭当前标签后激活相邻标签；关闭最后一个标签后回到最近文件主页。
- 页面缓存继续按 Worker `DocumentId` 隔离。渲染请求额外绑定发起时的 `ReaderState` 实例，即使用户在等待渲染时快速切换标签，迟到的旧标签位图也不会覆盖当前标签。
- Worker 意外退出并重启后，会依次重新打开所有仍存在的标签并恢复各自内存视图；单个文件恢复失败只关闭该失败标签，不影响其他标签。当前活动标签能够恢复时仍保持活动。
- 程序正常退出前遍历所有标签保存阅读状态，而不只保存最后一个活动标签。
- 新增 7 项多标签集成测试，覆盖多文档同时打开、独立视图与翻译上下文、单标签释放和末标签回主页、相同路径去重、Worker 重启恢复全部标签、迟到渲染隔离、后续标签适配命令和标签排序。
