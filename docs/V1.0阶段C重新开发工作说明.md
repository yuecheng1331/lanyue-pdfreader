# V1.0 阶段 C 重新开发工作说明

起点：阶段 B 提交 `f04e47e`  
目标：以成熟 PDF 阅读器的交互规律重新实现阅读模式，不复用本轮已回退的连续/双页实现。

## 问题复盘

此前的问题不是单独的页面渲染错误，而是多个状态源彼此竞争：

- 当前页既被翻页控件、滚轮位置和渲染预取逻辑分别修改。
- 缩放同时影响渲染分辨率、页面尺寸、容器尺寸和滚动位置，导致双页左上对齐、跳动、闪烁及白屏。
- 连续模式依赖页码附近的渲染窗口，快速滚动到窗口外时出现空白或错误提示。
- 多页布局未使用统一的命中测试与选择坐标，造成除普通单页外文本选择缺失或错位。

后续实现必须以「一个文档坐标系、一个视口状态源」为前提，而不是继续修补页码同步。

## 建议的分阶段方案

### C1：阅读视图状态与坐标系（不改变用户功能）

- 定义单一阅读视图状态：阅读模式、缩放倍率、滚动偏移、可见区域。
- 页面使用 PDF 逻辑尺寸布局；位图只决定清晰度，不决定页面的显示几何尺寸。
- 当前页仅由视口内的可见页面计算，供页码显示和预加载使用；页码输入和翻页按钮只负责请求滚动到目标页面。
- 文本选择、批注高亮、鼠标命中和滚动可见性全部基于同一套页面坐标转换。

验收：不新增模式；在现有普通单页中切换适应页面、适应宽度和缩放时，页面不闪烁、不跳页，当前页显示稳定。

### C2：单页连续

- 使用纵向连续文档画布，所有页面按顺序放入同一布局。
- 右侧垂直滚动条与鼠标滚轮做连续、无级滚动；不得将滚轮换算为一次一次的翻页。
- 为高速滚动保留足够大的可见区前后缓冲；未渲染区域应显示正常的占位状态，不弹出“PDF 无法打开”类错误。
- Ctrl + 滚轮以鼠标指向的文档位置为锚点缩放；Shift + 滚轮支持连续横向平移。

验收：快速拖动滚动条、切换适应方式、缩放、切换标签后均可继续阅读，不白屏、不影响其他 PDF。

### C3：双页

- 双页是两个逻辑页面组成的一个纵向页面组；始终按两列排列，不提供横向连续方向切换。
- 初始进入双页时，适应页面按“整组宽度和高度”计算，保证左右页面均在可视区域内。
- 缩放只改变视口对固定逻辑画布的观察倍率，不根据缩放倍率重新排版或将页面贴到左上角。
- 放大后允许连续横向和纵向移动；缩小后页面组居中显示。

验收：任意缩放、适应页面和适应宽度后，左右页相对位置不变；不会只显示左页或出现蓝色选中背景。

### C4：文本选择与回归

- 将 C1 的统一坐标转换接入单页连续和双页。
- 确认选择、复制、高亮批注在所有已支持阅读模式下可用；不能因为临时选择页面而改写文档的阅读状态。
- 对多标签、恢复关闭标签、移动/缺失文件和 Worker 恢复做回归，确保失败文档被隔离或跳过，不阻断其余文档恢复。

验收：阅读模式、文本选择和多标签恢复组合使用时没有全局白屏、全局无法打开或串页现象。

## 不可违反的设计边界

- 不把 PDF 位图像素尺寸当作页面布局尺寸；高分辨率渲染只用于提高图像清晰度。
- 不在缩放操作中重新请求每一页的 PDF 渲染，除非用户显式要求提升渲染质量。
- 不把滚轮事件直接映射为页码加减；连续模式必须保留连续滚动偏移。
- 不用控件选中状态作为阅读状态，以免出现蓝色选中框。
- 不允许单一文档的渲染失败清空所有标签或阻断其他文档恢复。
- 在 C3 完成前不加入额外阅读模式、封面独立、横向双页连续等扩展功能。

## 测试策略

每个子阶段结束时再进行一次集中人工测试，阶段内不打断请求测试。自动验证至少覆盖：

- 页面组的布局尺寸和两页对齐；
- 缩放前后锚点对应的文档坐标保持稳定；
- 滚动偏移到当前页、当前页到预加载范围的计算；
- 多标签之间阅读视图状态隔离；
- 缺失文件或渲染失败时跳过失败文档，其余文档仍能恢复；
- 文本选择坐标在不同阅读模式、缩放倍率下正确映射。

完整测试套件如继续存在宿主退出挂起，应记录为测试基础设施问题，并至少提供构建结果和与本阶段相关的定向测试结果；不得将其表述为全量测试通过。
