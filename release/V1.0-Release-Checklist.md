# 澜阅 PDF V1.0 Release Checklist

## 1. Automated Release Gate

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Publish-WindowsX64.ps1
```

The script must complete all release gates:

- restore dependencies and run the full Release test suite;
- publish the WPF app and PDF Worker as a Windows x64 self-contained package;
- verify required runtime files, `pdfium.dll`, and `e_sqlite3.dll`;
- reject user data, settings, logs, credentials, symbols, and databases from the package;
- create `artifacts/LanyuePDF-1.0.0-win-x64.zip`;
- create the matching `.sha256` file and `release-manifest.json`;
- extract the ZIP and compare file size and SHA-256 for every file;
- launch the extracted app with isolated user data and disabled system .NET runtime lookup.

## 2. Artifact Checks

- ZIP filename and About window both show `1.0.0`.
- `release-manifest.json` has `"tests": "passed"`.
- `release-manifest.json` has `"dirtyBuild": false`.
- `release-manifest.json` has `"extractedArchiveVerified": true`.
- `release-manifest.json` has `"isolatedLaunchVerified": true`.
- The `.sha256` value matches a freshly calculated SHA-256 hash.
- The ZIP does not contain `reader.db`, settings JSON, backup settings, logs, API keys, credentials, certificates, or debug symbols.
- The extracted app starts without installing .NET or copying extra native libraries.

## 3. Windows 11 x64 Manual Acceptance

- Start `LanyuePDF.exe` from a fresh extracted ZIP directory.
- Confirm startup lands on the recent-documents home page and does not auto-open a PDF.
- Open multiple PDFs and verify tab switching, tab ordering, tab closing, and independent reading state.
- Verify page navigation, continuous single-page reading, double-page reading, zoom, fit page, fit width, Ctrl+wheel zoom, and rotation.
- Verify text selection, cross-page selection, copy, search from selection, and Ctrl+F.
- Verify full-text search progress, cancellation, result navigation, and highlight clearing when the search panel is closed.
- Verify DeepSeek translation for selected text, current page, and page ranges.
- Verify translation cache hits, semantic segment cache, word translation shortcut path, history deletion, and comparison translation.
- Verify custom translation styles, editable prompt presets, restoreable built-in presets, and shortcut settings.
- Verify local annotations: highlight, note, color edit, delete, navigation, PDF write, Save As, Ctrl+S, and close-time unsaved-edit prompt.
- Kill `LocalPdfReader.PdfWorker.exe` and confirm worker restart plus open-tab recovery.
- Confirm user data is written only under `%LocalAppData%\LocalPdfReader`, not the extracted app directory.

## 4. Stage H Performance, Safety, And Release Gate

Stage H is the V1.0 performance, safety, and release phase. It is not only a packaging closeout. The release candidate must cover:

- Worker resource limits: bounded open-document count, bounded unreleased shared bitmaps, and rendered-page size validation before shared memory allocation.
- Large-document behavior: render cache stays bounded while sweeping hundreds or thousands of pages; manual acceptance should include a high-page-count PDF when available.
- Annotation save pressure: many pending local annotations can be converted to PDF write operations without duplicate identities, and stale LocalPdfReader-owned annotations are deleted.
- Database migration: V0.5 reading state, recent documents, local annotations, and session data survive the schema upgrade to V1.0 translation/cache tables.
- V0.5 upgrade acceptance: launch with a copied V0.5 user-data directory or representative migrated database and verify old reading state plus local annotations remain visible.
- Release gate: full Release tests, sensitive-file exclusion, archive extraction verification, isolated launch, and `dirtyBuild=false`.
- Clean Windows 11 acceptance: run the portable package from a clean extracted folder without relying on a developer build output.
- Final tag: create or align the local `v1.0.0` tag only after the accepted release commit and artifacts match.

## 5. Final Release Closeout

After manual acceptance:

1. keep the release commit and `v1.0.0` artifacts together;
2. keep `artifacts/LanyuePDF-1.0.0-win-x64.zip`, its `.sha256`, and `release-manifest.json` as the official release outputs;
3. do not push commits, tags, or release artifacts automatically unless explicitly requested.
